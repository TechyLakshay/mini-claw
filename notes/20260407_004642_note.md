**What are WebSockets?**
=======================

WebSockets provide a way for bidirectional, real-time communication between a client (typically a web browser) and a server over the web. They allow for efficient, low-latency exchange of data in both directions.

### Key Features

* **Bidirectional**: Allows both the client and server to send data to each other.
* **Real-time**: Enables near-instant communication between the client and server.
* **Efficient**: Uses a persistent connection, reducing the overhead of establishing new connections for each message.

### How WebSockets Work

1. The client initiates a WebSocket connection to the server.
2. The server accepts the connection and establishes a persistent, bidirectional channel.
3. Data can be sent from either the client or server through this channel at any time.
4. When data is received, it's delivered directly to the intended recipient, eliminating the need for polling.

### Use Cases

* **Live updates**: WebSockets enable real-time updates in applications like live chat, gaming, and social media.
* **Collaborative editing**: Multiple users can edit documents simultaneously using WebSockets.
* **Real-time analytics**: Servers can push data to clients in real-time, eliminating the need for polling.

**Example Use Case: Live Chat**
-----------------------------

```markdown
// Client-side code (JavaScript)
var socket = new WebSocket('ws://example.com/livechat');

socket.onmessage = function(event) {
  console.log(`Received message: ${event.data}`);
};

socket.send('Hello, server!');
```

```markdown
// Server-side code (Node.js with ws library)
const WebSocket = require('ws');
const wss = new WebSocket.Server({ port: 8080 });

wss.on('connection', function(ws) {
  console.log('Client connected');

  // Handle incoming messages from clients
  ws.on('message', function(message) {
    console.log(`Received message: ${message}`);
    // Send response back to client
    ws.send(`Hello, client!`);
  });
});
```
